# ADR-001

## Título

Migración gradual hacia la arquitectura "contable"

---

## Estado

Aceptado

---

## Contexto

La aplicación actual funciona utilizando la carpeta app/, pero la lógica se encuentra parcialmente distribuida entre app/ y src/.

Esta estructura dificulta el mantenimiento y genera duplicidad de responsabilidades.

---

## Decisión

Se construirá una nueva arquitectura denominada "contable".

Durante varios sprints convivirá con la arquitectura anterior.

La migración será progresiva.

No se moverá código sin antes verificar que los tests continúan funcionando.

---

## Consecuencias

Positivas

- Menor deuda técnica.
- Mejor organización.
- Mayor facilidad para incorporar nuevas funcionalidades.
- Mejor cobertura de pruebas.

Negativas

Durante algunos sprints coexistirán dos arquitecturas.
